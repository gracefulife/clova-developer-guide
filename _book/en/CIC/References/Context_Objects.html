Context information

Context information

Context information indicates various states of a client. To provide context, include context objects when sending event messages with the CIC APIs. Context information must reflect states at the time when user has spoken. The following are the available context objects.

  • AudioPlayer.PlaybackState
  • Device.DeviceState
  • Clova.FreetalkState
  • Clova.Location
  • Clova.SavedPlace
  • Speaker.VolumeState

AudioPlayer.PlaybackState

PlaybackState is a message format that contains information of media currently playing or previously played. This information can be sent to an extension that provides a music streaming service.

Message format

{
  "header": {
    "namespace": "AudioPlayer",
    "name": "PlaybackState"
  },
  "payload": {
    "offsetInMilliseconds": {{number}},
    "totalInMilliseconds": {{number}},
    "playerActivity": {{string}},
    "stream": {{AudioStreamObject}}
  }
}

Payload field

Field name Type Field description Required
offsetInMilliseconds number Recently played media's last playback point (offset). Unit is millisecond. You can leave the field empty when playerActivity is "IDLE". No
playerActivity string Indicates a player state. Available values are:
  • "IDLE": In an idle state
  • "PLAYING": In a playing state
  • "PAUSED": In a paused state
  • "STOPPED": In a stopped state
 Yes
stream AudioStreamObject An object that contains details of media currently playing. You can leave the field empty when playerActivity is "IDLE". Enter media details (stream object) returned from AudioPlayer.Play or AudioPlayer.StreamDelivered directive message. No
totalInMilliseconds number Total length of the media recently played. Unit is millisecond. You can leave the field empty when playerActivity is "IDLE". No

Message example

Example 1: When playback has stopped

// Case 1: When playback has stopped
{
  "header": {
    "namespace": "AudioPlayer",
    "name": "PlaybackState"
  },
  "payload": {
    "offsetInMilliseconds": 1,
    "totalInMilliseconds": 100,
    "playerActivity": "STOPPED",
    "stream": {
      "beginAtInMilliseconds": 0,
      "progressReport": {
        "progressReportDelayInMilliseconds": null,
        "progressReportIntervalInMilliseconds": null,
        "progressReportPositionInMilliseconds": 60000
      },
      "token": "TR-NM-17740107",
      "url": "clova:TR-NM-17740107"
    }
  }
}

Example 2: When a player is inactivated

{
  "header": {
    "namespace": "AudioPlayer",
    "name": "PlaybackState"
  },
  "payload": {
    "playerActivity": "IDLE"
  }
}

See also

  • AudioPlayer.Play
  • AudioPlayer.StreamDeliver
  • AudioPlayer.StreamRequested

Device.DeviceState

DeviceState is a message format that sends device state information, such as a current local time in a user's device.

Message format

{
  "header": {
    "namespace": "Device",
    "name": "DeviceState"
  },
  "payload": {
    "localTime": {{string}}
  }
}

Payload field

Field name Type Field description Required
localTime string Current time set in a client device (ISO 8601 format) Yes

Message example

{
  "header": {
    "namespace": "Device",
    "name": "DeviceState"
  },
  "payload": {
    "localTime": "2017-04-06T13:34:15.074361+08:28"
  }
}

See also

  • SpeechRecognizer.Recognize

Clova.FreetalkState

FreetalkState is a message format that shows a state of the Clova Freetalk service at a client side.

Message format

{
  "header": {
      "namespace": "Clova",
      "name": "FreetalkState"
  },
  "payload": {
      "foreground": {{boolean}},
      "reprompt": {{boolean}}
  }
}

Payload field

Field name Type Field description Required
foreground boolean Whether the Freetalk mode is activated or not
  • true: Activated.
  • false: Inactivated.
 Yes
reprompt boolean Whether reprompted for a user response or not. This state value verifies whether a user was reprompted for a response or not when the user did not give input for a specified time.
  • true: Reprompted for a user response and the Freetalk mode will be finished.
  • false: Did not reprompted for a user response and is waiting for a response.
 Yes

Remarks

The Freetalk mode is available only in English at the moment. If the foreground field is set to true, set the lang field of a Recognize event message of SpeechRecognizer API to "en".

Message example

{
  "header": {
      "namespace": "Clova",
      "name": "FreetalkState"
  },
  "payload": {
      "foreground": flase,
      "reprompt": true
  }
}

See also

  • SpeechRecognizer.Recognize

Clova.Location

Message format that sends client's current location information. You can obtain location information from GPS, base stations or network devices and send it to CIC.

Message format

{
  "header": {
    "namespace": "Clova",
    "name": "Location"
  },
  "payload": {
    "latitude": {{string}},
    "longitude": {{string}},
    "refreshedAt": {{string}}
  }
}

Payload field

Field name Type Field description Required
latitude string Latitude Yes
longitude string Longitude Yes
refreshedAt string The time when the location was last checked (UTC, ISO 8601 format) Yes

Message example

{
  "header": {
    "namespace": "Clova",
    "name": "Location"
  },
  "payload": {
    "latitude": "37.3594915",
    "longitude": "127.1032242",
    "refreshedAt": "2017-04-06T13:34:15.074361+08:28"
  }
}

See also

  • SpeechRecognizer.Recognize

Clova.SavedPlace

Message format that sends client's pre-saved location information.

Message format

{
  "header": {
    "namespace": "Clova",
    "name": "SavedPlace"
  },
  "payload": {
    "places": [
      {
        "latitude": {{string}},
        "longitude": {{string}},
        "refreshedAt": {{string}}
        "name": {{string}}
      }
    ]
  }
}

Payload field

Field name Type Field description Required
places[] object array An object array that contains pre-saved location information Yes
places[].latitude string Latitude Yes
places[].longitude string Longitude Yes
places[].name string The name of the location saved. Available values are:
  • 회사
  • 집
 Yes
places[].refreshedAt string The time when the location was saved (UTC, ISO 8601 format) Yes

Message example

{
  "header": {
    "namespace": "Clova",
    "name": "SavedPlace"
  },
  "payload": {
    "places": [
      {
        "latitude": "37.3594915",
        "longitude": "127.1032242",
        "refreshedAt": "2017-04-06T13:34:15.074361+08:28"
        "name": "집"
      },
      {
        "latitude": "36.3542315",
        "longitude": "125.1345242",
        "refreshedAt": "2017-03-12T10:21:33.089723+08:28"
        "name": "회사"
      }
    ]
  }
}

See also

  • SpeechRecognizer.Recognize

Speaker.VolumeState

Message format that contains speaker information at the time when a user has spoken, such as volume size or whether speaker is muted or unmuted.

Message format

{
  "header": {
      "namespace": "Speaker",
      "name": "VolumeState"
  },
  "payload": {
      "volume": {{number}},
      "muted": {{boolean}}
  }
}

Payload field

Field name Type Field description Required
muted boolean Whether muted or unmuted Yes
volume number Speaker's current volume size (0-10) Yes

Message example

{
  "header": {
      "namespace": "Speaker",
      "name": "VolumeState"
  },
  "payload": {
      "volume": 8,
      "muted": false
  }
}
